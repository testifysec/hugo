6. Federate the CI identity & go keyless terminal ci

Register the GitHub Actions OIDC identity the platform trusts for uploads (no long-lived secret), then the workflow build is keyless: cilock-action mints a fresh OIDC token, gets a short-lived Fulcio cert, signs, and stores in Archivista — signing material never reaches the build steps. That isolation earns L2 (hosted builder) and L3 (non-falsifiable provenance).

The workflow is deliberately minimal — everything except the attestor list derives from defaults (platform URL, keyless Fulcio, Archivista storage). The one non-obvious requirement is id-token: write:

permissions:
  contents: read
  id-token: write          # the only non-obvious requirement
jobs:
  attested-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@<sha>
      - uses: actions/setup-go@<sha>
        with: { go-version: '1.26' }
      - uses: aflock-ai/cilock-action@<sha>
        with:
          step: build
          command: go build -trimpath -o dist/hugo .
          attestations: environment git github slsa   # only override

The attestation it produces is keyless and tied to the build's own identity — no shared secret, no long-lived key:

7. Verification gate — close the loop ci

Producing an attestation isn't enough; the pipeline must act on it. A signed witness policy (supply-chain/gates/) pins the trusted signer to this workflow's own keyless identity — Fulcio CA TestifySec Platform Fulcio CA, OIDC issuer token.actions.githubusercontent.com, source repo testifysec/hugo, runner github-hosted. The build then verifies its own binary against that policy and fails closed if it doesn't match:

The policy itself is signed by the offline release key (Step 3); the attestation it trusts is the keyless CI identity. That separation — operator-signed policy, build-signed evidence — is the standard witness shape.
